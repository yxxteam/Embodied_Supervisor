# 仓库与目录说明

本文档是 `Embodied_Supervisor/README.md` 的目录级展开版，用来回答两个问题：

1. 每个子仓到底负责什么模块。
2. 每个目录为什么要存在，后续代码应落到哪里。

## 1. 根仓库定位

根仓库是“总览仓库”，不是把所有运行时代码揉在一起的单体仓库。

它负责：

- 保存整体架构图、目录说明和协作文档。
- 固化五个技术边界子仓的职责。
- 作为 GitHub 首页入口，让团队成员先理解边界，再进入各自负责目录。

Embodied_Supervisor/

## 3. 顶层目录说明

### `docs/`

根仓库文档中心。建议长期保留以下类型文档：

- 架构图与模块 API 图
- 仓库与目录说明
- GitHub 协作说明
- 分工说明和 teammate 指南
- 答辩 / 展示材料入口

当前关键文件：

- `embodied_supervisor_module_architecture.html`
- `仓库与目录说明.md`
- `仓库规划说明.md`
- `GitHub上传与协作指南.md`

### `edge-client-sdk/`

边缘监督执行面。负责：

- 设备接入
- 时间轴对齐
- 边缘侧任务监督
- 边缘到云端事件上送

不负责：

- 教师工作站页面
- 云端控制面 API
- 数据库存储实现

### `frontend-web-app/`

教师工作站前端。负责：

- 实时监督台
- 时间轴回放
- 语义搜索页面
- 教学卡片管理界面

不负责：

- 直接访问数据库
- 自行定义共享接口字段

### `backend-api-service/`

云端控制面。负责：

- 会话管理
- 边缘事件接入
- 复杂异常升级
- 教学资产导出
- 实时流转发与人工介入入口

不负责：

- 具体数据库连接细节
- 页面渲染逻辑

### `data-access-layer/`

持久化与检索底座。负责：

- PostgreSQL 元数据访问
- Redis 缓存与实时状态
- MinIO/S3 媒体对象访问
- Qdrant 向量检索

不负责：

- 控制器
- 页面逻辑
- 业务流程编排

### `shared-models-and-apis/`

中央契约仓库。负责：

- OpenAPI
- JSON Schema
- DTO
- Proto
- 示例 SOP / HTTP 请求响应

不负责：

- 运行时页面
- 运行时数据库访问
- 运行时检测或推理逻辑

## 4. 子仓内部目录约定

### `edge-client-sdk/`

```text
edge-client-sdk/
  README.md
  src/
    edge/
      capture/
      context/
      supervisor/
      uplink/
  tests/
  docs/
```

目录职责：

- `src/edge/capture/`
  - 摄像头、麦克风、机器人状态接入适配。
- `src/edge/context/`
  - 时间戳对齐、上下文快照、ring buffer。
- `src/edge/supervisor/`
  - `SOP/todo` 任务图谱、规则引擎、偏差识别。
- `src/edge/uplink/`
  - 事件上送、媒体上传、指导 / 复位下行桥。

### `backend-api-service/`

```text
backend-api-service/
  README.md
  src/
    api/
      ingest/
      guidance/
      realtime/
      assets/
  tests/
  docs/
```

目录职责：

- `src/api/ingest/`
  - `POST /api/v1/sessions`
  - `POST /api/v1/sessions/{id}/edge-events:batch`
  - `POST /api/v1/sessions/{id}/media`
- `src/api/guidance/`
  - 复杂异常升级、RAG 查询、VLM 调用。
- `src/api/realtime/`
  - SSE 状态流、人工介入入口、告警转发。
- `src/api/assets/`
  - 教学卡片生成、回放包导出、语义检索入口。

### `data-access-layer/`

```text
data-access-layer/
  README.md
  src/
    data/
      postgres/
      cache/
      object_store/
      vector_store/
  tests/
  docs/
```

目录职责：

- `src/data/postgres/`
  - 会话、事件、告警、教学卡片、导出任务。
- `src/data/cache/`
  - Redis 状态缓存、幂等键、Pub/Sub。
- `src/data/object_store/`
  - 原始视频、关键帧、片段、导出包。
- `src/data/vector_store/`
  - Embedding 写入与语义检索。

### `shared-models-and-apis/`

```text
shared-models-and-apis/
  README.md
  openapi/
  schemas/
    events/
    entities/
  proto/
  examples/
    sop/
    http/
  docs/
```

目录职责：

- `openapi/`
  - 控制面接口规范。
- `schemas/events/`
  - `DeviationEvent`、`TaskNodeProgress`、`GuidanceSuggestion` 等事件对象。
- `schemas/entities/`
  - `Session`、`TeachingCard`、`MediaObject` 等实体对象。
- `proto/`
  - 机器人状态流、边缘内部强类型消息。
- `examples/sop/`
  - 任务图谱 / `todo` 配置样例。
- `examples/http/`
  - 典型请求响应样例。

## 6. 当前目录状态说明

目前这些目录已经在仓库里建成了骨架，但多数还是占位目录。

这意味着：

- 文档和目录边界已经落地。
- 运行时代码还没有落地。
- 后续新增代码时，不应再回到“随便找个目录先塞进去”的方式。

## 7. 推荐下一步

按依赖顺序，最值得优先落地的是：

1. `shared-models-and-apis/openapi/` 下的首批控制面接口。
2. `shared-models-and-apis/schemas/events/` 下的核心事件对象。
3. `edge-client-sdk/src/edge/supervisor/` 下的 `todo/SOP` 监督链路。
4. `backend-api-service/src/api/realtime/` 下的实时状态流接口。
